# 篮球教案生成优化 - 2026-04-03

## 优化概述

本次优化解决了两个核心问题：
1. **缺少递进式关联**：热身环节没有体现与当天训练内容的关联
2. **缺少动作图解**：没有生成篮球战术图解式的SVG图片

## 核心改进

### 1. SVG战术图解标准

每个训练活动都会生成符合篮球战术图解标准的SVG图片，包含5大核心元素：

#### 视觉元素标准
- **学员位置**：蓝色圆圈（`#2563eb`），内部白色数字编号（1, 2, 3...）
- **教练位置**：红色方块（`#dc2626`），内部白色"教"字
- **球的位置**：橙色实心圆点（`#ea580c`）
- **动作路线**：红色箭头线条，带marker-end箭头标记
- **场地标志**：灰色虚线边框（`#374151`），尺寸400x300像素

#### 组织形式支持
- **双人训练**：两人面对面，显示传球/配合路线
- **分组训练（3人）**：每组3人三角形站位，显示球的位置和传球路线
- **排面训练**：多排多列整齐排列，教练在前方指导
- **依次训练**：学员纵队排列，依次进行
- **集体训练**：学员半圆形围绕教练

#### 示例SVG结构
```xml
<svg width="400" height="300" xmlns="http://www.w3.org/2000/svg">
  <defs>
    <marker id="arrow-red" markerWidth="10" markerHeight="10" refX="9" refY="3" orient="auto">
      <polygon points="0 0, 10 3, 0 6" fill="#dc2626" />
    </marker>
  </defs>
  <!-- 场地边框 -->
  <rect x="10" y="10" width="380" height="280" stroke="#374151" fill="none" stroke-width="2" stroke-dasharray="5,5"/>
  <!-- 教练 -->
  <rect x="190" y="80" width="20" height="20" fill="#dc2626" stroke="white" stroke-width="2"/>
  <text x="200" y="93" text-anchor="middle" font-size="10" fill="white" font-weight="bold">教</text>
  <!-- 学员1 -->
  <circle cx="150" cy="150" r="12" fill="#2563eb" stroke="white" stroke-width="2"/>
  <text x="150" y="155" text-anchor="middle" font-size="11" fill="white" font-weight="bold">1</text>
  <!-- 球 -->
  <circle cx="150" cy="150" r="4" fill="#ea580c"/>
  <!-- 路线 -->
  <line x1="165" y1="150" x2="235" y2="150" stroke="#dc2626" stroke-width="2" marker-end="url(#arrow-red)" stroke-dasharray="5,3"/>
</svg>
```

### 2. 递进式关联设计

每个教案必须包含完整的递进式关联设计，实现从热身到实战的完整链条。

#### 三层次递进结构

**第一层：热身环节（基础动作）**
- 在球性熟悉或动态伸展中，加入当天重点训练的基础版本
- 目的：让学员熟悉动作要领和球性，为正式训练做准备
- 示例：如果重点是"体前变向运球"
  - 热身球性：原地体前左右手交替运球（熟悉换手动作）
  - 热身动态伸展：持球体前绕环（活动肩关节）

**第二层：正式训练（标准动作）**
- 技术训练环节的标准练习
- 目的：掌握正确的技术规范和动作要领
- 示例：行进间体前变向运球（标准变向动作）
  - 增加难度：Z字形体前变向过障碍（增加移动和障碍）

**第三层：对抗比赛（实战应用）**
- 在实战情境中运用该技能
- 目的：提升在比赛情境下的运用能力和实战技巧
- 示例：3v3对抗中的体前变向突破（实战应用）

#### trainingProgression字段格式

```
本节课重点训练【技能名称】，递进式设计为：
1. 热身环节：通过【具体热身动作】（基础版本），让学员熟悉球性和基本动作要领，为正式训练做准备；
2. 正式训练：进行【具体技术训练】（标准动作），掌握正确的技术规范和动作要领；
3. 对抗比赛：在【具体对抗形式】中实战应用，提升在比赛情境下的运用能力和实战技巧。
实现从基础熟悉→标准掌握→实战应用的完整递进链条。
```

#### 具体关联示例

**示例1：重点训练 = 体前变向运球**
- 热身-球性熟悉：原地体前左右手交替运球（2组×30秒）
- 热身-动态伸展：持球体前绕环（1组×10次）
- 技术训练：行进间体前变向运球（3组×20米）
- 技术训练：Z字形体前变向过障碍（3组）
- 对抗比赛：3v3实战，强调体前变向突破（10分钟）

**示例2：重点训练 = 双手胸前传球**
- 热身-球性熟悉：原地抛接球（2组×10次）
- 技术训练：两人一组双手胸前传球（3组×20次）
- 技术训练：移动中双手胸前传球（3组×5分钟）
- 战术训练：二打一配合，重点传球时机（10分钟）
- 对抗比赛：4v4比赛，统计成功传球次数（15分钟）

### 3. 智能递进式设计生成

根据活动名称自动识别技能类型，并生成对应的3层次递进式说明。

#### 技能类型识别

**运球类**
- 体前变向运球："原地体前变向 → 行进间体前变向 → 障碍间快速变向"
- 胯下运球："原地胯下运球 → 行进间胯下运球 → 连续胯下变向"
- 高低运球："原地高低运球 → 行进间高低运球 → 高低运球变速突破"
- 默认："原地运球 → 行进间直线运球 → 变向运球突破"

**传球类**
- 双手胸前传球："原地双手胸前传球 → 移动中胸前传球 → 对抗中传球"
- 击地传球："原地击地传球 → 移动中击地传球 → 隔人击地传球"
- 默认："原地传球 → 移动中传球 → 对抗中传球"

**投篮类**
- 三步上篮："原地三步上篮脚步 → 持球三步上篮 → 运球三步上篮"
- 默认："近距离投篮 → 中距离投篮 → 移动中投篮"

**防守类**
- 滑步："原地防守姿势 → 慢速滑步 → 快速滑步跟随"
- 默认："原地防守姿势 → 一对一防守 → 实战防守"

**拉伸类**
- 默认："小幅度拉伸 → 标准幅度拉伸 → 动态拉伸"

## 技术实现

### 前端改动 (plan/new/page.tsx)

1. **SVG图解渲染**
```tsx
{activity.drillDiagram && (
  <div className="mt-3 bg-white rounded-lg border border-gray-200 p-3">
    <div className="text-xs font-medium text-gray-600 mb-2">动作路线示意图</div>
    <div 
      className="flex justify-center"
      dangerouslySetInnerHTML={{ __html: activity.drillDiagram }}
    />
  </div>
)}
```

2. **组数/次数/递进式显示**
```tsx
{(activity.sets || activity.repetitions || activity.progression) && (
  <div className="mt-3 grid grid-cols-1 gap-2 text-xs">
    {activity.sets && (
      <div className="flex items-center gap-2">
        <span className="font-medium text-gray-700">组数：</span>
        <span className="text-gray-600">{activity.sets}</span>
      </div>
    )}
    {/* ... */}
  </div>
)}
```

### 后端改动 (api/generate-plan/route.ts)

1. **AI Prompt增强**
   - 详细SVG生成标准（5大元素 + 示例代码）
   - 递进式关联3层次 + 具体示例
   - 严格格式要求（400x300像素、颜色标准、必须包含元素）

2. **SVG生成函数** `generateDefaultDrillDiagram()`
   - 支持5种组织形式（双人/分组/排面/依次/集体）
   - 每种形式有专门的布局算法
   - 包含图例说明

3. **智能递进式生成** `generateProgressionByActivity()`
   - 根据活动名称识别技能类型
   - 自动生成3层次递进式说明
   - 支持运球/传球/投篮/防守/拉伸等类型

4. **训练递进说明生成** `generateTrainingProgression()`
   - 分析各节活动内容
   - 自动提取重点技能
   - 生成完整的前后关联说明

5. **数据验证修复** `validateAndFixActivity()`
   - 确保所有必需字段存在
   - 修复缺失的SVG图解
   - 补充缺失的递进式说明
   - 智能生成组数和次数

## 使用指南

### 生成教案

1. 访问 `/plan/new` 页面
2. 选择年龄段、时长、场地等参数
3. **关键**：填写"重点训练技能"（如"运球"、"传球"）
4. 点击"AI生成教案"
5. 查看生成的教案，验证：
   - 每个活动是否有SVG图解
   - 是否有递进式设计（3层次）
   - trainingProgression是否包含完整关联说明

### 验证SVG图解

在生成的教案中，每个活动下方会显示"动作路线示意图"区域，检查：
- 是否有蓝色圆圈（学员）+ 数字编号
- 是否有红色方块（教练）+ "教"字
- 是否有橙色圆点（球）
- 是否有红色箭头路线
- 是否有场地边框

### 验证递进式关联

检查每个活动的"递进式设计"区域，确保包含3层次：
- 基础：简单版本（通常是原地动作）
- 进阶：增加难度（通常是移动中）
- 挑战：综合应用（通常是实战模拟）

检查教案底部的"训练递进说明"，确保包含：
- 本节课重点训练的技能名称
- 热身环节的具体关联动作
- 正式训练的具体内容
- 对抗比赛的具体形式
- 完整的前后关联链条

## 技术规格

### SVG标准
- **尺寸**：400x300像素
- **背景**：白色
- **格式**：标准SVG + XML命名空间
- **颜色**：严格按照标准色值
- **元素**：必须包含5大核心元素

### 递进式设计
- **层次**：必须3层次（基础→进阶→挑战）
- **格式**：使用"→"分隔，明确标注层次名称
- **关联**：trainingProgression必须包含完整说明
- **智能生成**：基于活动名称自动识别技能类型

### 数据格式

**TrainingActivity接口**
```typescript
interface TrainingActivity {
  // ... 其他字段
  
  // 新增/增强字段
  sets?: string;              // 组数（如："2-3组"）
  repetitions?: string;       // 次数/时间（如："每组30秒或20次"）
  progression?: string;       // 递进式设计（3层次）
  drillDiagram?: string;      // SVG图解（完整SVG代码）
}
```

**TrainingPlanOutput接口**
```typescript
interface TrainingPlanOutput {
  // ... 其他字段
  
  trainingProgression: string;  // 完整的前后关联说明
}
```

## 注意事项

1. **AI生成质量**：SVG图解和递进式设计依赖AI理解prompt，可能需要微调prompt以获得最佳效果
2. **自动修复**：如果AI生成的数据不完整，系统会自动修复（生成默认SVG和递进式）
3. **前端渲染**：SVG使用dangerouslySetInnerHTML渲染，确保SVG代码安全
4. **性能**：SVG图片在400x300像素内，文件大小控制在合理范围
