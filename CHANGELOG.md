## [5.8.1] - 2026-04-09

### 修复
- **教案生成 JSON 截断修复**：
  - Kimi 模型从 moonshot-v1-8k 升级到 moonshot-v1-32k，避免输出被截断
  - max_tokens 从 8000 提升到 16000
  - 新增 `tryRepairTruncatedJson()` 函数：检测未闭合字符串/数组/对象并智能补全
  - 即使 AI 输出被截断，也能尽量恢复可用 JSON

---

## [5.8.0] - 2026-04-09

### 优化
- **教案生成大幅优化**：
  - 热身环节优化：慢跑+拉伸严格控制在5分钟内（慢跑2分钟+拉伸3分钟）
  - 动作描述6要素：姿势、动作、发力、时间/次数、形式、要点目的
  - duration计算方法：确保标注时间与实际训练量匹配
  - 动作细节细化：运球（手/次数/高度/路线）、传球（手/方式/次数/距离）、投篮（方式/位置/数量）、防守（对象/移动/距离）
- **战术库扩充**：
  - 进攻战术：快攻类、阵地基础配合、整体阵地进攻、破防守专用、固定战术
  - 防守战术：人盯人、区域联防、紧逼夹击、协同轮转、个人防守
- **SVG图解规范**（教科书式移动符号）：
  - 学员位置：蓝色圆圈+编号
  - 教练位置：红色方块
  - 球位置：橙色圆点
  - 移动路线：实线箭头(运球)、虚线箭头(传球)、波浪线(跑位)、折线(变向)
  - 防守位置：红色小圆圈+D编号
  - 掩护位置：两条平行短线

### 代码质量
- 自动优化：移除未使用的导入、Prettier 格式化、TypeScript 严格化

## [5.6.1] - 2026-04-07

### 新增
- **支付 API 实现**：`/api/payments` 从空桩升级为完整 CRUD

  - 课程订单自动关联课时：支付成功后自动创建/更新课程购买记录
- **订单收款弹窗增强**：展示历史支付记录（支付方式、金额、操作人、日期）

### 优化
- 清理无用依赖：移除 `zustand`（未使用）和 `tencentcloud-sdk-nodejs-common`（语音识别已自行实现签名）
- 修正 Payment 接口字段名与后端一致（method→paymentMethod, transactionId→transactionNo）

### 体验
- Dashboard 个性化问候：头部展示教练头像、名字、基于时间的问候语、今日课程数
- MobileNav 真实未读数：从 `Math.random()` 改为调用 `/api/notifications` 获取真实未读通知数

## [5.6.0] - 2026-04-07

### 新增
- **通知系统全面升级**：
  - 通知服务层 `notificationService.ts`（训练完成通知、课时不足检查、课程到期检查、批量发送、统计）
  - 训练完成自动通知：训练结束后自动发送训练报告给家长（含教练评分和反馈）
  - 通知模板管理 Tab：查看/编辑/启停通知模板，支持 6 个默认模板
  - 批量发送通知：支持全选/多选学员，内容支持 {{playerName}} 变量替换
  - 通知统计面板：总通知数、今日/本周发送量、已读率、状态分布进度条、类型分布
  - 自动检查合并：课时不足 + 课程到期两个检查规则合并为一次调用
  - 防重复发送：24小时内不重复发送同类通知

### 改进
- notification-templates/[id] API 修复 params 为 Promise 类型（Next.js 15 规范）
- PUT 模板编辑权限从 admin-only 放宽为 coach 也可编辑
- 通知管理页面重构为三 Tab 布局：通知列表 / 通知模板 / 数据统计
